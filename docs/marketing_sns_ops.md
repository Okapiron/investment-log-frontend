# TradeTrace マーケ/SNS運用 Playbook (v1)

このドキュメントは、開発と並行してマーケティングを運用するための実務手順です。

## 0. 目的
- 初期公開フェーズで「認知 -> 訪問 -> 登録 -> 継続利用」の流れを作る。
- 開発速度を落とさず、運用データで改善優先度を決める。

## 1. 体制
- 開発レーン:
  - 機能実装、バグ修正、公開運用
- マーケレーン:
  - X運用、投稿設計、反応分析、改善提案
- 合流ルール:
  - 週1回、30分で「開発優先度」と「訴求テーマ」を同期する

## 2. 窓口方針（公開初期）
- 一次窓口: X DM
  - 軽い質問、使い方相談、不具合の一次報告
- 正式窓口: メールまたはフォーム
  - アカウント/データ削除、法務、個人情報関連
- 運用ルール:
  - X DMで受けた正式案件はメール/フォームへ誘導する
  - DMのスクショ転送ではなく、要点テキストで記録する

## 3. Xアカウント運用
### 3.1 プロフィール
- 何のサービスか（投資の振り返りログ）
- 誰向けか（初心者〜中級）
- 何ができるか（記録・チャート振り返り・レビュー）
- リンク（公開URL）

### 3.2 固定ポスト（必須）
- 30秒説明
- 3つの価値
- 使い始めるリンク
- 不具合窓口（DM/メール）

### 3.3 投稿の柱
- 使い方:
  - 新規記録、詳細編集、レビュー完了の流れ
- 改善ログ:
  - 何を直したか、何が楽になったか
- 学び:
  - 振り返りの観点、レビュー習慣化のコツ
- ユーザーの声:
  - 匿名化して紹介（許諾前提）

### 3.4 投稿頻度
- 週3〜5投稿
- 週1で振り返りレポート

## 4. KPI（初期）
- 認知:
  - Xプロフィールアクセス
  - 投稿インプレッション
- 獲得:
  - URLクリック数
  - 新規登録数
- 定着:
  - 初回Trade作成率
  - 7日継続率（最低1回ログイン）

## 5. サブエージェント運用仕様
### 5.1 役割
- 週次投稿計画の作成
- 投稿文案（日本語）とハッシュタグの提案
- KPI集計と改善案の提示
- 開発依頼候補の抽出（優先度付き）

### 5.2 入力
- 先週のKPI
- 今週のリリース内容
- 今週訴求したいテーマ
- 禁止事項（言えないこと）

### 5.3 出力
- 週次投稿カレンダー（5本）
- 投稿本文テンプレ
- KPIレビュー（良かった点/課題/次アクション）
- 開発バックログ候補（上位3件）

### 5.4 ガードレール
- 投資助言に見える断定表現を避ける
- 利回り保証表現を禁止
- ユーザーデータは匿名化前提
- 未実装機能を既存機能として書かない

## 6. 週次運用フロー（推奨）
1. 月曜:
   - KPI確認
   - 今週テーマ確定
2. 火曜:
   - 投稿カレンダー確定
3. 水〜金:
   - 投稿、DM一次対応
4. 金曜:
   - 週次レポート作成
   - 次週の開発優先度に反映

## 7. 緊急時対応
- 障害告知:
  - 15分以内に「調査中」告知
  - 復旧後に再発防止方針を共有
- 誹謗/荒らし:
  - 反応ルールを固定（削除/非表示/通報）
- 重大問い合わせ:
  - DMで詳細を受けず正式窓口へ移行

## 8. すぐ使えるサブエージェント指示テンプレ
以下を毎週そのまま渡して運用する。

```text
あなたはTradeTraceのマーケ運用担当です。以下を作成してください。

[入力]
- 今週のテーマ:
- 今週のリリース/変更点:
- 先週KPI:
- 言ってはいけないこと:

[出力要件]
1) 週次投稿カレンダー（5本）
2) 各投稿の本文案（日本語、短文）
3) KPIレビュー（良い点/課題/次アクション）
4) 開発に渡す改善提案3件（優先度付き）

[ルール]
- 投資助言の断定表現は禁止
- 未実装機能を実装済みと書かない
- ユーザー情報は匿名化
```
